01 · Introduction

Qu'est-ce que Ktor ?

Ktor est un framework open-source créé par JetBrains (les créateurs de Kotlin et IntelliJ IDEA). Il permet de construire des serveurs web et des clients HTTP en Kotlin avec une approche asynchrone basée sur les coroutines.

🔧

Modulaire par design

Vous n'installez que ce dont vous avez besoin. Pas de dépendances inutiles. Chaque fonctionnalité est un plugin optionnel.

⚡

100% Kotlin & Coroutines

Conçu pour tirer parti des coroutines Kotlin : code asynchrone lisible, pas de callbacks ni de reactive streams complexes.

🚀

Léger & Rapide

Démarrage ultra-rapide, faible empreinte mémoire. Idéal pour les microservices et les environnements cloud.

🧩

Client + Serveur

Ktor peut être utilisé côté serveur ET comme client HTTP multiplateforme (Android, iOS, Desktop, Web).

🔒

Type-safe

Grâce à Kotlin, vous bénéficiez de la sécurité des types à la compilation. Moins de bugs en production.

🌐

Multiplateforme

Le client Ktor tourne sur Kotlin/JVM, Kotlin/JS, Kotlin/Native. Une seule API pour tous vos projets.

02 · Installation

Installation & Configuration

Deux approches : utiliser le générateur officiel sur start.ktor.io (recommandé pour débuter) ou configurer manuellement Gradle.

Kotlin · build.gradle.kts 
// Plugins Gradle nécessaires
plugins {
    kotlin("jvm") version "2.0.0"
    id("io.ktor.plugin") version "3.0.0"
    kotlin("plugin.serialization") version "2.0.0"
}

group = "com.example"
version = "0.0.1"

application {
    mainClass.set("com.example.ApplicationKt")
}

dependencies {
    // ✅ Ktor Server (Netty comme moteur HTTP)
    implementation("io.ktor:ktor-server-netty")
    implementation("io.ktor:ktor-server-core")

    // ✅ Sérialisation JSON (kotlinx.serialization)
    implementation("io.ktor:ktor-server-content-negotiation")
    implementation("io.ktor:ktor-serialization-kotlinx-json")

    // ✅ Logs
    implementation("ch.qos.logback:logback-classic:1.5.6")

    // ✅ Tests
    testImplementation("io.ktor:ktor-server-test-host")
    testImplementation("org.jetbrains.kotlin:kotlin-test-junit")
}
💡

start.ktor.io est le moyen le plus simple pour démarrer. Il génère un projet complet avec les dépendances que vous choisissez via une interface graphique. Rendez-vous sur start.ktor.io, sélectionnez vos plugins (Routing, JSON, Auth…), cliquez sur "Generate" et importez le ZIP dans IntelliJ IDEA.

XML · pom.xml (extrait)
<!-- Dans <dependencies> -->

04 · Routing

Routing & Gestion des routes

Le routing Ktor permet de définir quelles fonctions répondent à quelles URLs. Il supporte les paramètres, les routes imbriquées et tous les verbes HTTP.

01

Routes de base (GET, POST, PUT, DELETE)

Chaque verbe HTTP a sa propre fonction dans le DSL Ktor. La lambda reçoit un objet ApplicationCall pour lire la requête et envoyer la réponse.

Kotlin · Routing.kt
fun Application.configureRouting() {
    routing {

        // GET  → Lire des données
        get("/users") {
            call.respondText("Liste de tous les utilisateurs")
        }

        // POST → Créer une ressource
        post("/users") {
            val body = call.receiveText()
            call.respond(HttpStatusCode.Created, "Créé : $body")
        }

        // PUT  → Modifier complètement
        put("/users/{id}") {
            val id = call.parameters["id"]
            call.respondText("Mis à jour l'utilisateur $id")
        }

        // DELETE → Supprimer
        delete("/users/{id}") {
            call.respond(HttpStatusCode.NoContent)
        }
    }
}
02

Paramètres de routes et query strings

Ktor permet d'extraire des paramètres dynamiques depuis l'URL ({id}) et des query strings (?search=…).

Kotlin
get("/users/{id}") {
    // Paramètre de route : /users/42 → id = "42"
    val id = call.parameters["id"]
        ?: return@get call.respond(HttpStatusCode.BadRequest)

    call.respondText("Utilisateur #$id")
}

get("/search") {
    // Query string : /search?q=Kotlin&limit=10
    val query = call.request.queryParameters["q"] ?: ""
    val limit = call.request.queryParameters["limit"]?.toInt() ?: 20

    call.respondText("Recherche '$query', max $limit résultats")
}
03

Routes imbriquées & groupement

Organisez vos routes avec des blocs route() pour éviter la répétition de préfixes et mieux structurer votre code.

Kotlin
routing {
    // Toutes les routes /api/v1
    route("/api/v1") {

        route("/users") {
            get { /* GET /api/v1/users */ }
            post { /* POST /api/v1/users */ }

            route("/{id}") {
                get { /* GET /api/v1/users/{id} */ }
                delete { /* DELETE /api/v1/users/{id} */ }
            }
        }

        route("/products") {
            get { /* GET /api/v1/products */ }
        }
    }
}
💡

Bonne pratique : Extrayez vos routes dans des fichiers séparés et utilisez des extension functions sur Route pour garder votre Application.kt propre.

05 · Requêtes & Réponses

Lire les requêtes, envoyer des réponses

Le cœur de Ktor est l'objet ApplicationCall disponible dans chaque handler. Il donne accès à la requête entrante et permet d'envoyer la réponse.

📥 Lire la requête

Kotlin
post("/example") {
    // Headers
    val token = call.request
        .headers["Authorization"]

    // Body texte brut
    val body = call.receiveText()

    // Body JSON (avec Content Negotiation)
    val user = call.receive<User>()

    // Multipart (upload fichier)
    val mp = call.receiveMultipart()
}

📤 Envoyer la réponse

Kotlin
get("/example") {
    // Texte simple
    call.respondText("Bonjour !")

    // JSON (avec sérialisation)
    call.respond(user)

    // Status code personnalisé
    call.respond(
        HttpStatusCode.Created,
        user
    )

    // Redirection
    call.respondRedirect("/autre-page")

    // HTML (avec plugin HTML DSL)
    call.respondHtml {
        body { p { +"Hello HTML" } }
    }
}

07 · JSON & Sérialisation

Sérialisation JSON avec Kotlinx

Ktor s'intègre avec kotlinx.serialization pour convertir automatiquement vos data classes Kotlin en JSON et vice-versa.

01

Configurer le plugin ContentNegotiation

Kotlin · plugins/Serialization.kt
fun Application.configureSerialization() {
    install(ContentNegotiation) {
        json(Json {
            prettyPrint = true        // JSON indenté
            isLenient = true          // JSON permissif
            ignoreUnknownKeys = true  // Ignore les champs inconnus
            encodeDefaults = true     // Inclut les valeurs par défaut
        })
    }
}
02

Créer un modèle de données

Annotez votre data class avec @Serializable pour que Kotlin puisse la convertir en JSON automatiquement.

Kotlin · models/User.kt
import kotlinx.serialization.Serializable

@Serializable
data class User(
    val id: Int,
    val nom: String,
    val email: String,
    val age: Int? = null   // champ optionnel
)

// Résultat JSON :
// {
//   "id": 1,
//   "nom": "Alice",
//   "email": "alice@example.com",
//   "age": 28
// }
03

Routes avec JSON automatique

Kotlin · routes/UserRoutes.kt
val users = mutableListOf(
    User(1, "Alice", "alice@ex.com", 28),
    User(2, "Bob",   "bob@ex.com",   32)
)

fun Route.userRoutes() {
    route("/users") {

        // Retourne la liste → [{"id":1,...}, {...}]
        get {
            call.respond(users)
        }

        // Reçoit du JSON et crée un User
        post {
            val newUser = call.receive<User>()
            users.add(newUser)
            call.respond(HttpStatusCode.Created, newUser)
        }

        // Retourne un seul User en JSON
        get("/{id}") {
            val id = call.parameters["id"]?.toIntOrNull()
            val user = users.find { it.id == id }
                ?: return@get call.respond(HttpStatusCode.NotFound)
            call.respond(user)
        }
    }
}
08 · Authentification

Authentification JWT

Ktor supporte plusieurs méthodes d'authentification. La plus courante pour les APIs REST est le JWT (JSON Web Token).

⚠️

Ajoutez la dépendance JWT : implementation("io.ktor:ktor-server-auth") et implementation("io.ktor:ktor-server-auth-jwt")

Kotlin · plugins/Security.kt — Configuration JWT complète
fun Application.configureSecurity() {
    val secret = "mon-secret-jwt-très-long"
    val issuer = "http://localhost:8080"
    val audience = "mon-api"

    install(Authentication) {
        jwt("auth-jwt") {
            realm = "Accès sécurisé"
            verifier(
                JWT.require(Algorithm.HMAC256(secret))
                    .withAudience(audience)
                    .withIssuer(issuer)
                    .build()
            )
            // Valide le token et extrait le principal
            validate { credential ->
                if (credential.payload.audience.contains(audience))
                    JWTPrincipal(credential.payload)
                else null
            }
            challenge { _, _ ->
                call.respond(
                    HttpStatusCode.Unauthorized,
                    "Token invalide ou expiré"
                )
            }
        }
    }
}

// ---- Générer un token lors du login ----
post("/login") {
    val user = call.receive<LoginRequest>()
    // Vérifier les credentials en base de données...

    val token = JWT.create()
        .withAudience(audience)
        .withIssuer(issuer)
        .withClaim("userId", user.id)
        .withExpiresAt(Date(System.currentTimeMillis() + 86_400_000))
        .sign(Algorithm.HMAC256(secret))

    call.respond(hashMapOf("token" to token))
}

// ---- Protéger des routes avec authenticate{} ----
authenticate("auth-jwt") {
    get("/profil") {
        // Accessible uniquement avec un token valide
        val principal = call.principal<JWTPrincipal>()
        val userId = principal?.payload?.getClaim("userId")?.asInt()
        call.respondText("Profil de l'utilisateur #$userId")
    }
}
09 · Client HTTP

Client HTTP Ktor

Ktor fournit aussi un puissant client HTTP pour appeler des APIs externes depuis votre application. Il utilise les coroutines et supporte de nombreux moteurs.

Kotlin · Utilisation du client HTTP
import io.ktor.client.*
import io.ktor.client.engine.cio.*
import io.ktor.client.request.*
import io.ktor.client.statement.*
import io.ktor.client.call.*

// 1. Créer le client (avec CIO comme moteur)
val client = HttpClient(CIO) {
    install(ContentNegotiation) {
        json()  // Active la désérialisation JSON
    }
    install(HttpTimeout) {
        requestTimeoutMillis = 5000
    }
}

// 2. GET simple
val response: HttpResponse = client.get("https://api.exemple.com/users")
val texte = response.bodyAsText()

// 3. GET avec désérialisation automatique en data class
val users: List<User> = client.get("https://api.exemple.com/users").body()

// 4. POST avec body JSON
val created: User = client.post("https://api.exemple.com/users") {
    contentType(ContentType.Application.Json)
    setBody(User(0, "Claire", "claire@ex.com"))
}.body()

// 5. Headers & Auth
val data = client.get("https://api.exemple.com/private") {
    headers {
        append(HttpHeaders.Authorization, "Bearer $token")
        append("X-Custom-Header", "valeur")
    }
}.body<MyData>()

// 6. Toujours fermer le client quand vous n'en avez plus besoin
client.close()
10 · Tests

Tester votre application Ktor

Ktor facilite les tests en fournissant un moteur de test spécial qui ne démarre pas de vrai serveur HTTP — vos tests sont ultra-rapides.

Kotlin · ApplicationTest.kt
import io.ktor.client.request.*
import io.ktor.client.statement.*
import io.ktor.http.*
import io.ktor.server.testing.*
import kotlin.test.*

class ApplicationTest {

    @Test
    fun `GET users retourne 200`() = testApplication {
        // Configure l'application à tester
        application {
            configureRouting()
            configureSerialization()
        }

        // Envoie une requête test
        val response = client.get("/users")

        // Vérifie la réponse
        assertEquals(HttpStatusCode.OK, response.status)
        assertContains(response.bodyAsText(), "Alice")
    }

    @Test
    fun `POST users crée un utilisateur`() = testApplication {
        application { module() }

        val response = client.post("/users") {
            contentType(ContentType.Application.Json)
            setBody("""{"id":99,"nom":"Test","email":"t@t.com"}""")
        }

        assertEquals(HttpStatusCode.Created, response.status)
    }

    @Test
    fun `Route protégée retourne 401 sans token`() = testApplication {
        application { module() }

        val response = client.get("/profil")
        assertEquals(HttpStatusCode.Unauthorized, response.status)
    }
}
11 · Déploiement

Déployer votre application Ktor

Ktor peut être déployé de plusieurs façons. Le plus courant est de créer un Fat JAR autonome.

01

Créer un Fat JAR (JAR autonome)

Le plugin Ktor Gradle permet de créer un JAR qui contient tout (application + dépendances). Il suffit d'un JDK pour le lancer.

Bash
# Compiler et empaqueter
./gradlew buildFatJar

# Lancer le JAR produit
java -jar build/libs/mon-app-all.jar

# Avec des variables d'environnement
PORT=8080 java -jar build/libs/mon-app-all.jar
02

Dockeriser l'application

Dockerfile
# Build stage
FROM gradle:8.8-jdk21 AS build
WORKDIR /app
COPY . .
RUN gradle buildFatJar

# Runtime stage (image légère)
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=build /app/build/libs/*-all.jar app.jar

# Exposer le port
EXPOSE 8080

# Lancer l'application
ENTRYPOINT ["java", "-jar", "app.jar"]
Bash · Build & Run Docker
docker build -t mon-app-ktor .
docker run -p 8080:8080 mon-app-ktor
12 · Comparaison

Ktor vs Spring Boot vs Micronaut

Quand choisir Ktor plutôt qu'un autre framework JVM ?

Critère Ktor Spring Boot Micronaut
Langage principal ✓ Kotlin-first Java (supporte Kotlin) Java (supporte Kotlin)
Démarrage à froid ✓ Très rapide (~100ms) Lent (1-5s) Rapide (~300ms)
Empreinte mémoire ✓ Très légère Lourde Légère
Courbe d'apprentissage Douce (si vous connaissez Kotlin) Modérée (annotations) Modérée
Coroutines natives ✓ Oui, nativement — Réactif via WebFlux — Partiellement
Écosystème / plugins Ciblé, croissant ✓ Très riche Riche
Multiplateforme ✓ Client KMP — JVM uniquement — JVM / GraalVM
Idéal pour Microservices, APIs, KMP Grandes applis entreprise Serverless, GraalVM
🎯

Notre recommandation : Choisissez Ktor si vous développez en Kotlin, que vous voulez un serveur léger et rapide, ou que vous ciblez des microservices. Pour une grande application existante en Java, Spring Boot reste le roi de l'écosystème.
